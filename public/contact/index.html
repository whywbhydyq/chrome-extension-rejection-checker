Contact and issue reporting

Last updated: June 6, 2026

Use this page for questions about the Chrome Extension Rejection Checker site, scanner behavior, documentation errors, broken links, accessibility issues, or reports that appear inaccurate for a Manifest V3 package. The preferred workflow is to describe the issue without attaching private extension source code.

Do not send private ZIP files, extension source code, API keys, OAuth client secrets, Chrome Web Store credentials, production certificates, customer data, or unpublished business logic. The scanner is designed to run locally in your browser, so most debugging can be done from the finding ID, file type, manifest field, and a short redacted snippet.

What to include in a useful report

  • The page URL where you saw the problem.
  • The browser and operating system used for the local scan.
  • The finding ID, severity, and short message from the report.
  • The relevant manifest field or file category, such as background service worker, content script, CSP, permissions, or web accessible resource.
  • A minimal redacted example if the issue depends on syntax. Remove domains, secrets, tokens, customer names, and proprietary implementation details.

What this contact channel can and cannot do

The maintainer can review scanner bugs, false positives, false negatives, unclear wording, source documentation drift, and site accessibility problems. This contact channel cannot accelerate Chrome Web Store review, overturn a rejection, interpret private reviewer correspondence as an official representative, or provide legal advice about privacy policy wording. For a rejected item, continue to use the official Chrome Web Store Developer Dashboard and policy documentation as the source of record.

Response expectations

This is a free independent tool, so no service-level agreement is offered. High-value reports are precise, reproducible, and limited to the minimum detail needed to improve the scanner or page guidance. Reports that include sensitive code or credentials may be ignored or deleted for safety.
