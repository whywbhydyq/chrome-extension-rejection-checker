Disclaimer and review limitations

Last updated: June 6, 2026

Chrome Extension Rejection Checker provides a static, browser-based preflight scan for common Chrome extension package risks. It is an independent tool and is not affiliated with Google, Chrome, Chromium, or the Chrome Web Store review team. The site does not represent official policy decisions, approval likelihood, reviewer intent, or enforcement outcomes.

No approval guarantee

A clean scan does not guarantee Chrome Web Store approval. Chrome Web Store review may consider policy compliance, single purpose, user data handling, listing metadata, screenshots, privacy disclosures, support information, account history, runtime behavior, remote service behavior, reviewer judgment, and facts that are not visible inside a submitted ZIP file. The scanner only checks static package signals that can be inspected locally.

No complete security review

The report is not a security audit, penetration test, privacy impact assessment, legal review, or compliance certification. It can identify suspicious code patterns such as remote hosted code references, dynamic code execution, broad permissions, missing files, or risky CSP values. It cannot prove that an extension is safe, compliant, or free of vulnerabilities.

No legal or policy advice

Guidance on this site is provided for general developer education. You remain responsible for interpreting Chrome Web Store policies, preparing privacy disclosures, choosing permissions, documenting data use, and deciding whether to submit or resubmit an extension. For legal obligations, consult qualified counsel. For official Chrome Web Store requirements, consult Google’s current documentation and Developer Dashboard notices.

False positives and false negatives

Static scanning can be conservative. A finding may flag a pattern that is harmless in your architecture, or it may miss a risk hidden behind minification, code generation, string concatenation, obfuscation, server-side behavior, conditional loading, or runtime-only state. Use the report as a starting point for human review, not as the final authority.

Developer responsibility

  • Verify every finding against your actual extension behavior.
  • Review official Chrome extension and Chrome Web Store documentation before making submission decisions.
  • Keep your submitted ZIP, manifest, privacy disclosures, permissions, and store listing consistent.
  • Do not upload or send private source code to third parties unless you have authority to disclose it.
