Manifest and package checks

The scanner verifies that the ZIP contains a root manifest, uses Manifest V3, resolves declared script and icon paths, and does not look like a packaging mistake where the extension files were nested under an extra folder.

Remote code and dynamic execution

The report flags script loaders, workers, module imports, worklets, WebAssembly loading, eval, new Function, string timers, and CSP patterns that can trigger Manifest V3 review failures.

Permissions and privacy review

Broad host permissions, sensitive Chrome APIs, externally connectable declarations, and data access patterns are highlighted so you can align the package with your Chrome Web Store privacy disclosures.

How to use the report before resubmission

Start with critical findings that can block review outright, then handle high and medium warnings that may require code bundling, permission narrowing, or clearer privacy disclosures. A clean report does not guarantee approval because Chrome Web Store review also considers behavior, policy context, store listing accuracy, user consent flows, and reviewer interpretation. It does give you a repeatable local checklist for the package-level mistakes that are easy to miss when you are focused on product code.

If the scanner reports remotely hosted code, move executable JavaScript and WebAssembly into the extension package. If it reports dynamic code execution, replace string execution with static modules or explicit control flow. If it reports broad host permissions, consider optional permissions, narrower match patterns, or a stronger user-facing reason for the requested access.

Frequently asked questions

Does this tool upload my ZIP?

No. The scan runs locally in your browser. Your extension ZIP is read on your device, and the tool does not need a server-side upload to inspect package paths and text files.

Can this guarantee Chrome Web Store approval?

No. It is an independent static preflight scanner. Use it to catch common rejection risks before submission, then review the official Chrome Web Store policies and your actual extension behavior.
